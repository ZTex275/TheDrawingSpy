# Рисовальный шпион

Мобильная игра для компании на **.NET MAUI** (Android). Один телефон передаётся по кругу: каждый игрок видит загаданное слово, один из игроков — шпион и слова не знает. После обсуждения и рисования команда выбирает исход раунда, начисляются баллы.

## Требования

| Компонент | Версия |
|-----------|--------|
| .NET SDK | 10.0+ |
| Workload | `maui-android` |
| JDK | 17 (Microsoft OpenJDK или аналог) |
| Android SDK | API 36, Build-Tools 36 |
| Устройство | Android 5.0+ (API 21+) или эмулятор |

Для разработки в **Cursor / VS Code** достаточно .NET SDK и Android SDK. Visual Studio не обязательна.

## Настройка окружения

### 1. Установить .NET SDK и workload MAUI Android

```powershell
dotnet --version
dotnet workload install maui-android
```

### 2. Установить JDK 17

```powershell
winget install --id Microsoft.OpenJDK.17
```

### 3. Установить Android SDK

SDK по умолчанию ставится в `%LOCALAPPDATA%\Android\Sdk`.

Если SDK ещё нет, можно установить зависимости через MSBuild:

```powershell
$env:JAVA_HOME = "C:\Program Files\Microsoft\jdk-17.0.19.10-hotspot"
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"

dotnet build -t:InstallAndroidDependencies `
  -f net10.0-android `
  -p:AndroidSdkDirectory="$env:LOCALAPPDATA\Android\Sdk" `
  -p:JavaSdkDirectory="$env:JAVA_HOME" `
  -p:AcceptAndroidSdkLicenses=True
```

### 4. Переменные среды (рекомендуется)

| Переменная | Пример значения |
|------------|-----------------|
| `JAVA_HOME` | `C:\Program Files\Microsoft\jdk-17.0.19.10-hotspot` |
| `ANDROID_HOME` | `%LOCALAPPDATA%\Android\Sdk` |
| `ANDROID_SDK_ROOT` | `%LOCALAPPDATA%\Android\Sdk` |

В `PATH` добавьте:

```
%LOCALAPPDATA%\Android\Sdk\platform-tools
```

Проверка:

```powershell
adb devices
```

Должно отображаться подключённое устройство или эмулятор.

## Запуск проекта

### Через Cursor / VS Code (F5)

В проекте настроены `.vscode/launch.json` и `.vscode/tasks.json`:

- **F5** — сборка и запуск на подключённом Android-устройстве
- **Ctrl+Shift+B** — та же команда как задача сборки

### Через командную строку

```powershell
$env:JAVA_HOME = "C:\Program Files\Microsoft\jdk-17.0.19.10-hotspot"
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"

dotnet build -t:Run -f net10.0-android `
  -p:AndroidSdkDirectory="$env:LOCALAPPDATA\Android\Sdk" `
  -p:JavaSdkDirectory="$env:JAVA_HOME"
```

### Только сборка (без установки на устройство)

```powershell
dotnet build -f net10.0-android
```

## Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                      App (singleton)                     │
│                   App.Game → GameService                 │
└──────────────────────────┬──────────────────────────────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                 ▼
   ┌───────────┐    ┌────────────┐    ┌─────────────┐
   │  Pages    │    │  Helpers   │    │  Services   │
   │  (UI)     │◄──►│ Navigation │    │ GameService │
   │           │    │  Confirm   │    │ GameStorage │
   └───────────┘    └────────────┘    │  WordList   │
                                       └──────┬──────┘
                                              │
                                       Preferences (JSON)
```

- **GameService** — единый источник состояния игры (игроки, раунд, шпион, слово, баллы).
- **GameStorage** — сериализация снимка игры в `Preferences` (ключ `last_saved_game`).
- **NavigationPage** — линейная навигация между экранами; для критичных переходов стек намеренно перестраивается через `NavigationHelper`.

## Экраны приложения

| Экран | Файл | Назначение |
|-------|------|------------|
| **Настройка** | `SetupPage` | Число игроков (3–15), раундов (1–100), имена, старт / продолжение |
| **Раунд** | `RoundIntroPage` | Номер раунда, правила, кнопка «Начать раздачу» |
| **Раздача** | `RevealPage` | По очереди каждый видит слово или роль шпиона |
| **Итог раунда** | `OutcomePage` | Выбор исхода, overlay «Секреты раунда» |
| **Таблица** | `ScoreboardPage` | Баллы по раундам, итоги, следующий раунд |
| **Имена** | `PlayersPage` | Редактирование имён во время игры |

### Меню на экране раунда

- **Таблица** — просмотр текущих баллов
- **Имена** — изменить имена игроков
- **Новая игра** — сброс с подтверждением

## Игровой процесс

```
SetupPage
    │
    ├─► [Продолжить игру] ──► RoundIntroPage  (если есть незавершённая игра)
    │
    └─► [Начать игру] ──────► RoundIntroPage
                                  │
                                  ▼
                            RevealPage
                         (игрок 1 → 2 → … → N)
                                  │
                                  ▼
                            OutcomePage
                         (выбор исхода раунда)
                                  │
                                  ▼
                         «Секреты раунда»
                         (шпион + слово)
                                  │
                                  ▼
                           ScoreboardPage
                                  │
                    ┌─────────────┴─────────────┐
                    ▼                           ▼
           [Следующий раунд]              [Новая игра]
                    │                           │
                    ▼                           ▼
             RoundIntroPage                SetupPage
```

### Настройка (`SetupPage`)

- **Количество игроков** — ввод с клавиатуры или степпер (3–15).
- **Количество раундов** — ввод с клавиатуры или степпер (1–100).
- **Имена** — по умолчанию «Игрок 1», «Игрок 2», …; можно изменить до старта.
- При открытии подставляются **настройки последней игры**.
- Если игра не завершена — кнопка **«Продолжить игру (раунд X из Y)»**.

### Раздача ролей (`RevealPage`)

1. Экран «Передайте телефон: *Имя*».
2. Игрок нажимает **«Показать слово»** (кнопка неактивна 0,7 сек — защита от случайного нажатия).
3. Обычный игрок видит **загаданное слово**, шпион — **«Вы — ШПИОН!»**.
4. **«Скрыть и передать дальше»** — переход к следующему игроку.
5. После всех игроков — экран выбора исхода.

> Тап по всему экрану отключён — только явные кнопки.

### Итог раунда (`OutcomePage`)

Три варианта исхода (см. [баллы](#правила-начисления-баллов)).

После выбора — overlay **«Секреты раунда»** с именем шпиона и загаданным словом, затем переход к таблице.

## Правила начисления баллов

| Исход | Кто получает баллы |
|-------|-------------------|
| **Шпиона нашли** | Все, **кроме** шпиона → по **1** баллу |
| **Шпиона определили неверно** | Только шпион → **1** балл |
| **Шпион отгадал слово** | Только шпион → **2** балла |

Таблица показывает баллы за каждый раунд (Р1, Р2, …) и сумму (Σ). Ячейки, где игрок был шпионом, подсвечены. Лидер(ы) выделяются зелёным.

## Сохранение игры

Игра автоматически сохраняется на устройстве при:

- старте новой игры;
- завершении каждого раунда;
- изменении имён игроков.

**Что сохраняется:**

- количество игроков и раундов;
- имена игроков;
- текущий номер раунда;
- таблица баллов по раундам;
- список шпионов по раундам.

**Хранение:** `Microsoft.Maui.Storage.Preferences`, JSON, ключ `last_saved_game`.

**Продолжение:** на экране настройки, если `CurrentRound > 0` и не все раунды сыграны.

**Новая игра:** прогресс сохраняется в настройках, но текущая партия сбрасывается. Подтверждение — кастомное окно в стиле «Секреты раунда».

## Навигация и защита от «Назад»

Чтобы случайное нажатие «Назад» не ломало игру:

| Экран | Поведение «Назад» |
|-------|-------------------|
| `RevealPage` | Заблокировано |
| `OutcomePage` | Заблокировано |
| `ScoreboardPage` (после раунда) | Заблокировано |
| `ScoreboardPage` (просмотр из меню) | Кнопка «Закрыть» |

При переходе к таблице после раунда экраны раздачи и исхода **удаляются из стека навигации**, чтобы нельзя было вернуться на «сломанный» экран выбора исхода.

## Словарь слов

Файл `Services/WordList.cs` — около **360** русских слов (животные, еда, предметы, природа, спорт, профессии, фантастика и др.).

Слово и шпион выбираются **случайно** в начале каждого раунда (`GameService.StartNextRound`).

Чтобы добавить слова — допишите строки в массив `WordList.All`.

## Сборка APK

Release-сборка:

```powershell
dotnet publish -f net10.0-android -c Release `
  -p:AndroidSdkDirectory="$env:LOCALAPPDATA\Android\Sdk" `
  -p:JavaSdkDirectory="$env:JAVA_HOME"
```

APK будет в каталоге `bin/Release/net10.0-android/publish/`.

## Краткая справка для игроков

1. Настройте игроков и раунды, нажмите **«Начать игру»**.
2. На экране раунда — **«Начать раздачу слов»**.
3. Передавайте телефон по очереди; каждый смотрит своё слово и скрывает экран.
4. Рисуйте, обсуждайте, определите шпиона.
5. Выберите исход раунда на телефоне.
6. Посмотрите **«Секреты раунда»** и таблицу баллов.
7. **«Следующий раунд»** — игра продолжается до конца.

Приятной игры!
